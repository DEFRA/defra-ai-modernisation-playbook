The reverse engineering process produces several artefacts, from intermediate curated content through to the final Product Requirements Document. This section describes each output — what it is, how it's produced, what it contains, and what good looks like.

## Outputs

### Intermediate artefacts

- [HTML Mockups]({{ "/pages/output-reference/html-mockups/" | relative_url }}) — semantic HTML representations of legacy application screens
- [Curated Transcripts]({{ "/pages/output-reference/curated-transcripts/" | relative_url }}) — cleaned stakeholder interview transcripts with off-topic content removed

### Analysis outputs

- [Domain Analysis]({{ "/pages/output-reference/domain-analysis/" | relative_url }}) — strategic Domain-Driven Design analysis of the problem space
- [Interaction Analysis]({{ "/pages/output-reference/interaction-analysis/" | relative_url }}) — screen inventory, workflows, and navigation paths
- [Application Analysis]({{ "/pages/output-reference/application-analysis/" | relative_url }}) — legacy source code analysis covering architecture, business rules, and integrations
- [Database Analysis]({{ "/pages/output-reference/database-analysis/" | relative_url }}) — database schema, stored procedures, and data-level business rules

### Final deliverable

- [Product Requirements Document]({{ "/pages/output-reference/prd/" | relative_url }}) — the comprehensive PRD synthesised from all four analysis outputs
