# Project Directory Structure

The plugin expects a specific directory layout in the host project (the project you run Claude Code from). Some directories contain inputs that you provide, whilst others are populated by the plugin during analysis.

## Input directories

These directories must be set up before running the plugin.

| Directory | Contents |
|-----------|----------|
| `screenshots/` | UI screenshots of the legacy application |
| `transcripts/` | Stakeholder interview transcripts (.txt) |
| `src/` | Legacy application source code |

## Output directories

These directories and files are generated by the plugin.

| Directory | Contents | Produced by |
|-----------|----------|-------------|
| `output/` | Analysis files and PRD | Analyst agents and product-manager |
| `output/html/` | Semantic HTML mockups | `image-to-html` skill |
| `output/transcripts/` | Curated transcripts | `curate-transcript` skill |

## Full directory tree

```
project/
├── screenshots/          # Input: UI screenshots
│   ├── home.png
│   ├── search.png
│   └── ...
├── transcripts/          # Input: interview transcripts
│   ├── demo_walkthrough.txt
│   └── ...
├── src/                  # Input: legacy source code
│   ├── Solution.sln
│   └── ...
└── output/               # Generated: all outputs
    ├── html/             # HTML mockups
    │   ├── home.html
    │   ├── search.html
    │   └── ...
    ├── transcripts/      # Curated transcripts
    │   ├── demo_walkthrough_curated.txt
    │   └── ...
    ├── domain-analysis.md
    ├── interaction-analysis.md
    ├── application-analysis.md
    ├── database-analysis.md
    └── PRD.md
```

## Recommended `.gitignore` entries

The intermediate outputs can be regenerated from the raw inputs, so you may wish to exclude them from version control:

```gitignore
# Plugin intermediate outputs (regeneratable)
output/html/
output/transcripts/
```

## What to commit vs what to regenerate

- **Commit**: `output/PRD.md` and the four analysis files (`domain-analysis.md`, `interaction-analysis.md`, `application-analysis.md`, `database-analysis.md`) — these are the deliverables.
- **Regenerate**: `output/html/` mockups and `output/transcripts/` curated transcripts — these are intermediate artefacts that can be reproduced from the raw inputs.
